# E0017

Warning name: `ambiguous_loop_argument`

The usage of identifier is ambiguous. If it refers to the loop variable, please
use `as <id>` to bind it in the pattern. If it refers to the original value of
the variable before entering the loop, please bind it to a new binder outside
the loop.

```{note}
This warning is tied to the deprecated `loop` syntax. Current MoonBit also
reports `deprecated_syntax` for that construct, so this compatibility page has
no checked source example.
```

## Suggestion

Rewrite old `loop` code with a `for` loop when possible. For example:

```moonbit
let text : StringView = "asdf"
for i = 0; i < text.length(); i = i + 1 {
  println(text[i:])
}
```

If you are maintaining existing `loop` code, make the intended binding explicit:
use `as <id>` when you mean the loop variable, or bind the original value to a
different name before entering the loop.
