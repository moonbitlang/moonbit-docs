# E4198

Compiler diagnostic name: `struct_constr_not_implemented`.

When legacy code declares a constructor signature inside a `struct`, that
constructor must be implemented separately. The declaration only gives the
constructor signature; it does not create the implementation body.

```{literalinclude} /sources/error_codes/4198_error/top.mbt
:language: moonbit
```

## Suggestion

Define the constructor with the current top-level `Type::Type` form.
