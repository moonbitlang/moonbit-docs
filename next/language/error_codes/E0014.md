# E0014

Warning name: `alert` or `alert_<category>`

Alert warnings are emitted when code uses an API that the provider marked with
an alert attribute, such as [`#internal`](../attributes.md#internal-attribute).
The alert category is chosen by the API provider, so the warning name can be
`alert` for all alerts or `alert_<category>` for one category.

The provider module marks `experimental` as internal, and the importing module

when the dependency intentionally exposes that API for your use case.
